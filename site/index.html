Know when Claude

A traffic light in your system tray — visible from any window. Give Claude a task, switch away, and glance back: red means it's waiting, green means it's done.

watching your sessions…

Red

Claude is waiting for your input — a permission prompt or question is open.

Orange

Claude is working, or a session is sitting idle.

Green

Task finished — come back and collect your work.

White

No active Claude sessions right now.

Running several sessions? Any session that needs you wins the light. Otherwise the most recently active one decides — so a stale window can't hide your fresh green.

How it works

Two small parts. No network.

Hooks record state, a tiny tray app reads it. That's the whole thing.

01 — Hooks

Lifecycle events, captured

The plugin's hooks fire on Claude Code events — prompt submitted, tool running, permission needed, response finished — and write one tiny state file per session to ~/.claude/semaphore/.

02 — Tray app

A single Go binary

A static ~5 MB binary polls that directory once a second and shows the aggregate light. Menu offers Reset to idle and Quit. Sessions untouched for 12 hours are ignored, so a crash can't wedge the light.

No telemetry, no network. It reads local files, and nothing else.

Install

Two commands, inside Claude Code

Start a new session afterwards and the light appears in your tray within seconds. First run downloads the tray app for your platform (~5 MB) and registers it to start at login.

claude code
›/plugin marketplace add TaulantSela/claude-code-semaphore
›/plugin install semaphore@claude-code-semaphore

Requires curl — preinstalled on macOS/Linux, and ships with Git Bash on Windows (which Claude Code already needs there).
